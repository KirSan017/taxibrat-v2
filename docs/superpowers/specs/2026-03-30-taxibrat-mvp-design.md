# ТаксиБрат MVP — Спецификация дизайна

## 1. Обзор продукта

**ТаксиБрат** — сервис-аудитор таксопарков, помогающий водителям такси находить честные условия аренды авто. Данные о парках собираются вручную («тайными покупателями»), каждый парк получает математически обоснованный рейтинг.

**Бизнес-модель:** Скрытый лид — название лучшего парка скрыто до момента верификации водителя и обращения к менеджеру.

**Целевая аудитория:** Водители такси, арендующие автомобили у таксопарков.

## 2. Технический стек

| Компонент | Технология |
|-----------|-----------|
| Frontend + Backend | Next.js 15 (App Router) + TypeScript |
| Стилизация | Tailwind CSS + shadcn/ui |
| БД | PostgreSQL через Supabase |

**Обоснование:** Один кодбейз, AI хорошо генерирует Next.js, zero-cost деплой, mobile-first PWA, SEO из коробки.

## 4. Модель данных

### User
- id, phone, telegramId, name, role (DRIVER / MANAGER / ADMIN)
- verificationStatus (PENDING / APPROVED / REJECTED)
- rejectionReason, driverClass, experience
- photoLicense, photoSelfie (URL в Supabase Storage)

### TaxiPark
- id, internalName (видно только админам/менеджерам), city
- createdBy (менеджер), auditorComment, contractScan
- isActive

### ParkOffer (Предложение парка по классу авто)
Один парк → несколько офферов. Все 40+ полей из PRD, разбитых на 6 групп:

**Финансы:** rentPrice, schedule (7/0, 6/1, 5/2, 2/2), deposit, initialPayment, dailyPayment, depositReturnDays, parkCommission, withdrawalType (instant/daily/weekly), withdrawalFee.

**Авто:** brand, model, year, fuelType (petrol/lpg/cng), transmission (auto/manual), mileage, conditionScore (1-10), hasAC, hasHeatedSeats, hasBluetooth.

**Обслуживание:** maintenancePayer (park/driver/split), tirePayer, repairPayer, freeWash (bool), replacementCar (bool).

**Юридическое:** insuranceType (regular/taxi), hasKasko, hasWaybills, contractType (rent/rentToBuy/employment), hasLicense.

**Токсичность:** earlyReturnPenalty, territoryLimit, dailyMileageLimit, dirtPenalty, hiddenFees (text), managementScore (1-10).

**Бонусы:** hasBranding, hasChildSeat, ownDevice, homeStorage, minExperience, citizenship (rf/cis/any), aggregators (text[]), hasParking.

### RatingWeight
- paramName, weight (%), city
- Сумма весов по городу = 100%

### Rating (кеш)
- parkOfferId, totalScore (0-5), breakdown (JSON)

### VerificationRequest
- userId, status (PENDING/APPROVED/REJECTED), reviewedBy, reviewedAt, rejectionReason

## 5. Механика рейтинга

### Алгоритм

Не все поля ParkOffer участвуют в рейтинге. Поля-фильтры (schedule, brand, model, fuelType, transmission, contractType, citizenship, aggregators) используются только для фильтрации в каталоге. Рейтинг считается по числовым и булевым параметрам, у которых есть вес в RatingWeight.

1. **Нормализация** — для каждого числового параметра с весом > 0: min/max по всем офферам города и класса авто.
   - «Чем меньше, тем лучше» (цена, залог, штрафы): `score = (max - value) / (max - min)`
   - «Чем больше, тем лучше» (год авто, состояние): `score = (value - min) / (max - min)`
   - Булевы: 1 или 0
   - Если min === max: все получают score = 1 (нет разницы по параметру)

2. **Взвешивание** — каждый нормализованный балл × вес параметра (из RatingWeight).

3. **Финальная оценка** — сумма взвешенных баллов × 5 = оценка 0–5.

### Когда пересчитывается

- При добавлении/редактировании ParkOffer → пересчёт всех офферов города + класса
- При изменении весов → пересчёт всех офферов города
- Синхронный пересчёт (для MVP при <100 парках — миллисекунды)

## 6. Путь водителя (User Flow)

### Шаг 1: Крючок (Landing) — без авторизации
- Строка поиска «Введите название вашего таксопарка» (autocomplete из базы)
- Топ-3 парков города: рейтинг виден, названия скрыты, ключевые плюсы показаны
- Если парк не найден: «Мы пока не проверили этот парк, но вот лучшие варианты» → каталог

### Шаг 2: «Горькая правда» — без авторизации
- Карточка текущего парка: крупный рейтинг (напр. 3.1/5)
- Список параметров, где парк проигрывает рынку
- Кнопка «Показать варианты с рейтингом 4.5+»

### Шаг 3: Каталог — без авторизации
- Список офферов, отсортированный по рейтингу
- Фильтры: класс авто, цена (от/до), с залогом / без залога
- Названия парков скрыты → «Парк №XX»
- Видны: рейтинг, цена аренды, год авто, ключевые плюсы
- Кнопка «Узнать название и условия»

### Шаг 4: Регистрация + Верификация — барьер
- Telegram Login Widget (без SMS)
- Мини-анкета: имя, в каком классе работает
- Загрузка фото: права (лицо/оборот) + селфи с авто
- Статус «Документы на проверке»

### Шаг 5: Одобрение
- Менеджер одобряет в админке → Telegram-бот отправляет водителю:
  - Контакты выбранного парка
  - Приглашение в чат с менеджером
- При отклонении → бот отправляет причину

### Шаг 6: Чат с куратором
- Прямой чат в Telegram с менеджером проекта
- Менеджер аргументирует рейтинг данными аудита

## 7. Админка (Desktop)

### 7.1 Дашборд
- Счётчики: всего водителей, в очереди на проверку, парков в базе, новых за неделю
- Топ-5 популярных классов авто
- Последние заявки на верификацию

### 7.2 Аудит парков
- Таблица парков с поиском и фильтрами
- Форма добавления/редактирования: 40+ полей на 6 вкладках (Финансы, Авто, Обслуживание, Юридическое, Токсичность, Бонусы)
- Поле «Комментарий аудитора» + загрузка скана договора
- Автопересчёт рейтинга при сохранении

### 7.3 Настройка рейтинга (только Админ)
- Ползунки весов для каждого параметра (сумма = 100%)
- Предпросмотр: как изменятся топ-10 при новых весах
- Кнопка «Применить» → пересчёт

### 7.4 Модерация водителей
- Очередь заявок: фото документов, данные анкеты
- Кнопки «Одобрить» / «Отклонить» (с выбором причины из списка)
- При одобрении → автоматическое уведомление через Telegram-бот

## 8. Telegram-бот

Бот выполняет 3 функции:
1. **Авторизация** — Telegram Login Widget на сайте
2. **Уведомления** — одобрение/отклонение верификации, контакты парка
3. **Чат** — переход в диалог с менеджером

Бот НЕ дублирует функциональность сайта. Поиск, каталог, фильтры — только на сайте.

## 9. Scope MVP

### Входит
- Landing-крючок с поиском парка (autocomplete)
- Экран «Горькая правда»
- Каталог парков (скрытые названия, фильтры)
- Авторизация через Telegram Login Widget
- Верификация (загрузка фото, модерация)
- Telegram-бот (уведомления + контакты)
- Админка: CRUD парков, настройка весов, модерация, дашборд
- Рейтинговая система
- 1 город (Москва)
- Mobile-first адаптивный дизайн

### Не входит (на потом)
- Система внутренних баллов/валюты
- Встроенный чат (используем Telegram)
- SMS-авторизация
- Мульти-город UI (структура БД готова)
- PWA-установка
- Push-уведомления через браузер
- Отзывы водителей о парках
- Аналитика переходов
